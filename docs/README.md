```
1️⃣ 프로젝트 개요 읽기

   └─ 사용 기술과 구조 파악
   
3️⃣ 개발 워크플로우 숙지

4️⃣ API & DB 설계 확인
   └─ API.md와 DATABASE.md로 현재 구조 파악
   
5️⃣ 개발 진척도 및 다음 단계 확인
   └─ 현재 어디까지 왔고, 무엇을 해야 하는지 확인
   
6️⃣ 실제 코드 탐색
   └─ 프로젝트 루트의 CLAUDE.md 참고
```

### 최근 완료 항목

- ✅ Docker Compose 인프라 구축
- ✅ JWT 기반 인증 시스템
- ✅ 포트폴리오 CRUD 및 목표 비중 설정
- ✅ 금융상품 검색 및 관리, 포트폴리오 그룹
- ✅ 거래 원장 (복식부기, 매수/매도/입금/출금/배당)
- ✅ 평가 엔진 (포지션/시가평가/손익)
- ✅ 성과 분석 (TWR, CAGR, Volatility, MDD, Sharpe)
- ✅ 포트폴리오 비교 (Compare API + ECharts)
- ✅ 백테스트 엔진 (정적배분 + 리밸런싱 + 결과 차트)
- ✅ 리밸런싱 도구 (현재 vs 목표 비중 + 매매 추천)
- ✅ 다국어 지원 (한국어/영어)
- ✅ API 레퍼런스 및 데이터베이스 설계 문서

## 🔄 문서 업데이트 규칙

### 언제 업데이트하나요?

| 문서 | 업데이트 시점 | 필수 여부 |
|------|--------------|----------|
| **개발 진척도** | 주요 기능 완료, 테스트 추가 | ✅ 필수 |
| **다음 단계** | Sprint 완료, 우선순위 변경 | ✅ 필수 |
| **API 레퍼런스** | API 엔드포인트 추가/변경 | ✅ 필수 |
| **데이터베이스 설계** | 테이블 추가/수정, 인덱스 변경 | ✅ 필수 |
| **프로젝트 개요** | 기능 범위 변경, MVP 우선순위 변경 | 🟡 권장 |
| **기술 스택** | 새로운 라이브러리 추가, 아키텍처 변경 | 🟡 권장 |
| **워크플로우** | 개발 프로세스 변경 | 🟢 필요시 |

### 누가 업데이트하나요?

- **개발 진척도 & 다음 단계**: 모든 개발자 (기능 완료 시)
- **API 레퍼런스**: Backend 개발자 (API 변경 시)
- **데이터베이스 설계**: Backend 개발자 (스키마 변경 시)
- **프로젝트 개요**: PM, Tech Lead
- **기술 스택**: Tech Lead, Senior Developer
- **워크플로우**: Tech Lead (프로세스 변경 시)

### 문서 작성 가이드

**마크다운 컨벤션:**
- 제목: `#` ~ `###` 사용
- 강조: `**굵게**`, `*기울임*`
- 코드: \`인라인\`, \`\`\`블록\`\`\`
- 이모지: 섹션 구분에만 사용
- 표: 정보 비교에 활용

**문서 구조:**
```markdown
# 제목

> 한 줄 설명

---

내용...

---

**작성일:** YYYY-MM-DD
```
